## 3. What Was Undefined and Why

`DC` — the data cache object — was `undefined` when `loadHome()` ran.

`DC` is declared at line 2869 as `var DC = {};` inside the main `<script>` block. The most likely cause: a race condition or script execution order where `loadHome()` was called (via `initApp` → `loadPage` → `loadHome`) before the `var DC = {};` declaration was evaluated, OR the variable was somehow lost in the Vercel serverless function bundling context.

While the exact mechanism is unclear (the code *looks* correct), the fix is clear: guard against `DC` being undefined at the point of use.

## 4. Fix Applied

### loadHome — before:
```javascript
async function loadHome(){
  clearError('pg-home');
  try {
    if(!DC['pg-home']){
      // ...
    }
    var d=DC['pg-home'].d, sl=DC['pg-home'].sl, ar=DC['pg-home'].ar, sp=DC['pg-home'].sp;
```

### loadHome — after:
```javascript
async function loadHome(){
  clearError('pg-home');
  try {
    if(!DC) DC={};
    if(!DC['pg-home']){
      // ...
    }
    var c=DC['pg-home']||{};
    var d=c.d||null, sl=c.sl||null, ar=c.ar||null, sp=c.sp||null;
```

Two fixes:
1. `if(!DC) DC={};` — re-initialize if somehow undefined
2. `var c=DC['pg-home']||{};` — defensive destructuring so even if the cache entry is undefined, we get null values instead of a crash
